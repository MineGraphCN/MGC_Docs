常用格式

标题

# 文档标题 ## 二级标题 ... ##### 五级标题

在章节后加空格并接 {id="xxx"} 可以设置该章节的跳转 ID，可以在链接中使用。

标题中的西文字符会自动设置为跳转 ID，若没有西文则设置为编译之后的 UUID，每次编译都会变化。

手动设置的 ID 无西文要求，可以任意设置，仅当你会在其他地方（无论是站内还是站外）跳转到这个位置时才为它们设置 ID。

XML / HTML 代码

环绕插入行内代码 框选内容, CtrlAltT, T

<代码>内容</代码>

环绕插入代码块 框选内容, CtrlAltT, B

<代码> 内容块 </代码>

如果不包含具体内容，可以使用 <代码>。
快捷键仅 XML 行有效，无法在 Markdown 行中使用。

几乎所有 XML 块都可以像标题那样设置 ID，但是跳转位置不一定精确。

                      注释和待办事项（TODO）

                      新起一行并使用

                      [//]: # (注释内容)

                      一个板块应当在完成之后再取消注释

                            你应该（Should do）

                            • 给自己的文档起一个恰当的名字；

                            • 语言简洁明了，尽量让文字能直接表述你的观点；

                            • 引用参考资料时在文尾附上明确的来源。

                            你不应该（Should NOT do）

                            • 将文档上传至错误的分区；

                            • 不按照正确格式编写或使用不准确的文本；

                            • 对于没有确定的结论，不加以说明就编写进入文档；

                            • 未经管理组和其他执笔者同意，擅自变更由其他执笔者编写的文档内容。

                            你不能（MUST NOT do）

                            • 编写无关内容和违反社区相关规定的内容；

                            • 在文档内插入无关内容和违反社区相关规定的内容；

                            • 未经授权搬运、冒名使用他人研究成果；

                            • 在没有事实考究的情况下，轻易编写文档；

                            • 随意、轻易更改客观事实内容和已经定型的内容。
